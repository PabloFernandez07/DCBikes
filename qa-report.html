Bugs encontrados — 14 incidencias

BUG #1
precio coste (cost_price) accesible públicamente por cualquier visitante
Crítico
El campo cost_price (precio de compra al proveedor) de la tabla products se devuelve en la API REST de Supabase sin autenticación. Usando la anon key pública, cualquier visitante puede consultar el margen exacto de la tienda.
GET /rest/v1/products?select=id,name,cost_price,retail_price → {"cost_price":3000.00,"retail_price":6000.00}
Añadir política RLS de columna en Supabase: REVOKE SELECT (cost_price) ON products FROM anon; o excluir la columna del select público.
BUG #2
Sin HTTPS — credenciales de admin viajan en texto plano
Crítico
La aplicación solo sirve HTTP en el puerto 8100. Las credenciales del panel de administración (email + contraseña) se transmiten sin cifrado. El dominio canónico configurado en el código (https://dcbikescantabria.es) es actualmente inalcanzable.
curl -I http://34.155.113.218:8100/ → no Strict-Transport-Security, no Location a HTTPS
Configurar TLS con Let's Encrypt en nginx. Apuntar dcbikescantabria.es a la IP del servidor y habilitar HTTPS con redirección automática desde HTTP.
BUG #3
Aviso legal incompleto — incumplimiento LSSI art. 10
Crítico
La página /aviso-legal contiene cuatro marcadores de texto literal visibles para cualquier visitante: NIF/CIF pendiente, teléfono pendiente, forma jurídica pendiente, Registro Mercantil pendiente. Publicar una web sin estos datos infringe la Ley 34/2002 LSSI y puede acarrear sanciones.
"pendiente — añadir NIF o CIF" / "pendiente — añadir teléfono de contacto" / "pendiente — p. ej. Autónomo / S.L. / S.A." (extraído del bundle JS)
El cliente debe aportar: NIF/CIF, teléfono, denominación jurídica y número de inscripción registral. Rellenar antes de publicar el dominio.
BUG #4
Google Maps API Key expuesta y sin restricciones de dominio
Alto
La API Key de Google Places está hardcodeada en el bundle JS público. Fue probada directamente contra la API y confirmada activa (devuelve valoración 4,9 estrellas, 36 reseñas de DC Bikes). Sin restricciones de referrer, cualquier tercero puede usarla y generar facturación contra la cuenta.
AIzaSyDOfSC-6_HBplaUv8N1Im_3acun9ESd4vs — validada contra Places API v1, devuelve datos reales del negocio
En Google Cloud Console → Credentials → restringir la key a referrers dcbikescantabria.es/* y 34.155.113.218/*. Considerar proxy via Edge Function.
BUG #5
Sin compresión gzip/brotli — bundle de 1,4 MB sin comprimir
Alto
El servidor nginx no devuelve cabecera Content-Encoding. El bundle JS de 1,4 MB se transfiere sin comprimir a cada visitante. Con brotli se reduciría a ~350 KB (75% ahorro), mejorando el Time to Interactive especialmente en móvil.
curl -H "Accept-Encoding: gzip, deflate, br" → Content-Length: 1467263, sin Content-Encoding
En nginx.conf: gzip on; gzip_types text/javascript application/javascript text/css application/json; gzip_comp_level 6;
BUG #6
Sin Cache-Control en assets — re-descarga en cada visita
Alto
Ningún asset (JS, CSS, imágenes) tiene cabecera Cache-Control. Solo se envía ETag. Los assets tienen nombres hasheados (index-BFg6vJvU.js), lo que hace seguro el cacheo permanente, pero nginx no lo configura.
HTTP response headers: ETag presente, sin Cache-Control en todos los endpoints
location /assets/ { add_header Cache-Control "public, max-age=31536000, immutable"; }
BUG #7
Sin headers de seguridad HTTP
Alto
El servidor no envía ninguno de los headers de seguridad estándar: X-Frame-Options (riesgo clickjacking), X-Content-Type-Options, Content-Security-Policy, Referrer-Policy, Permissions-Policy.
curl -I → cero security headers presentes
Añadir en nginx: add_header X-Frame-Options "SAMEORIGIN"; add_header X-Content-Type-Options "nosniff"; add_header Referrer-Policy "strict-origin-when-cross-origin";
BUG #8
console.log internos de arquitectura en producción
Medio

BUG #9
Meta tags SEO no visibles a crawlers (SPA sin SSR)
Medio
El HTML estático solo tiene <title>DC Bikes Cantabria</title>. Todas las meta OG, Twitter Card y canonical se inyectan por React en runtime. Bots de redes sociales (WhatsApp, Facebook) y crawlers sin JS no verán descripción ni imagen de preview.
curl http://34.155.113.218:8100/ → HTML sin meta description, sin og:image, sin canonical
Añadir meta description y og:image básicos en el index.html estático como fallback. Para SEO completo a largo plazo, considerar pre-rendering con vite-plugin-ssr o migración a Next.js.

BUG #11
Rutas 404 devuelven HTTP 200 (soft 404)
Medio
Cualquier ruta inexistente (ej. /ruta-inexistente) devuelve HTTP 200 con el HTML shell. React renderiza el componente 404, pero Google lo indexa como página válida (soft 404), diluyendo el crawl budget.
curl -o /dev/null -w "%{http_code}" http://34.155.113.218:8100/ruta-inexistente → 200
Difícil de resolver con SPA pura sin SSR. Alternativa: añadir canonical a la home en el 404 component + noindex meta tag para que Google no indexe 404s.
BUG #12

Bajo
La anon key de Supabase es pública por diseño, pero su seguridad depende íntegramente de que RLS esté correctamente configurada en todas las tablas. Dado que cost_price ya es accesible (Bug #1), se recomienda auditoría completa de políticas RLS.
JWT anon extraído del bundle — confirma acceso a REST API
Auditar todas las tablas: products, categories, quotes, settings, analytics — verificar que anon solo puede leer lo que debe leer.
